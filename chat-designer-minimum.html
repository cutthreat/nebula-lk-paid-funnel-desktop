Базовый мессенджер

Левый список чатов, активный тред, header, разделители дат, входящие/исходящие сообщения, примеры read/delivered/sending.

Не надо тратить дизайн-слот на новый каркас чата.

Уже есть

Медиа и действия

Сетка изображений, видео-плитка, модалка видео на весь экран, аудиосообщения, состояние ответа, реакции, контекстное меню, сохраненные/поиск.

Нужны только состояния ошибок, не полный редизайн.

Нужно дорисовать для запуска

Нужно

1. Офер / оплата / кредиты

  • выбранный эксперт и формат консультации
  • пакет / credit offer
  • цена и что пользователь получает
  • оплатить / купить кредиты / пополнить
  • обработка, успех, ошибка, ожидание, отмена

Пользователь должен понимать, что именно покупает, до старта платного чата.

Нужно

2. Статус платной сессии

  • бесплатный предпросмотр / платный живой чат / ожидание / завершено
  • остаток кредитов или минут
  • лейбл стоимости / пакета
  • Действия: пополнить / продолжить / завершить чат

Главная защита от непонимания оплаты внутри чата.

Нужно

3. Вопрос перед чатом

  • выбранный эксперт / тема / пакет
  • поле вопроса
  • необязательный контекст
  • заметка о приватности и валидация

Без этого у эксперта нет структурированной задачи.

Нужно

4. Обрыв связи / плохая связь

  • эксперт потерял соединение
  • у клиента нестабильная связь
  • индикатор переподключения
  • списание на паузе
  • выбор: ждать / асинхронный ответ / поддержка

Если у эксперта плохая связь, клиент должен видеть, что услуга защищена и кредиты не списываются молча.

Нужно

5. Эксперт недоступен / задерживается

  • не в сети / занята после выбора
  • доступные альтернативные эксперты
  • офер бесплатных минут или бонуса, если используется
  • срок ожидания или состояние ожидания
  • отправить асинхронный / выбрать другого эксперта
  • подсказка по возврату или поддержке, если услуга не исполнена

Нельзя строить запуск на допущении, что эксперт всегда онлайн.

Нужно

6. Подключение к эксперту / без списания

  • выбранный эксперт
  • прогресс подключения
  • ожидаемое время, например 15-20 секунд, если это правда
  • явно: во время подключения списания нет
  • ждать / выбрать другого эксперта / обратиться в поддержку

Запись показывает это как отдельное состояние доверия: клиент не должен думать, что его уже тарифицируют во время подключения.

Нужно

7. Мало кредитов / нет кредитов

  • предупреждение о низком балансе
  • состояние остановленной сессии
  • пополнить и продолжить
  • завершить и сохранить стенограмму
  • предложение автопополнения с пропуском / обычной оплатой

Самое критичное прерывание в платном чате.

Нужно

8. Ошибка оплаты / другой способ

  • сообщение, что оплата не прошла
  • выбранный пакет / краткий итог заказа
  • попробовать другой способ
  • выбрать другой пакет
  • безопасный возврат в чат или поддержку

Это критичное для выручки состояние: покупатель не должен застрять на продолжении консультации.

Нужно

9. Ошибка отправки / загрузки

  • сообщение отправляется / ошибка / повтор
  • вложение загружается / ошибка / повтор / удалить
  • неподдерживаемый файл

Достаточно набора состояние компонентов, не отдельные большие страницы.

Нужно

10. Консультация завершена / стенограмма

  • консультация завершена
  • стенограмма сохранена
  • краткое содержание вопроса и финальный ответ
  • следующий вопрос, оценка, сохранить в закладки

Закрывает платное обещание и создает следующий заказ.

Нужно

11. Поддержка / возврат / жалоба

  • пожаловаться на сообщение / эксперта
  • запросить помощь / возврат
  • отправлено / на рассмотрении / решено

Нужен понятный путь безопасности и поддержки внутри платного чата.

Не просим сейчас
