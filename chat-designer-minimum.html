Уже есть, не просим заново

Covered

Базовый messenger

Левый список чатов, активный тред, header, date separators, входящие/исходящие сообщения, read/delivered/sending examples.

Не надо тратить дизайн-слот на новый shell.

Covered

Медиа и действия

Image grid, video tile, fullscreen video modal, audio messages, reply state, reactions, context menu, saved/search.

Нужны только failure states, не полный redesign.

Covered

Эксперт в контексте

Есть modal/profile card и вариант с expert sidebar.

Можно переиспользовать в paid chat flow.

Нужно дорисовать для запуска

Required

1. Offer / payment / credits

  • selected expert and format
  • package/credit offer
  • price and what user receives
  • Pay / Buy credits / Top up
  • processing, success, failed, pending, cancelled

Пользователь должен понимать, что именно покупает, до старта платного чата.

Required

2. Paid session status bar

  • free preview / paid live / waiting / ended
  • credits or minutes left
  • cost/package label
  • Top up / Continue / End chat CTA

Главная защита от непонимания оплаты внутри чата.

Required

3. Question intake

  • selected expert/topic/package
  • question textarea
  • optional context
  • privacy note and validation

Без этого у эксперта нет структурированной задачи.

Required

4. Connection lost / bad connection

  • expert connection lost
  • client connection unstable
  • reconnecting indicator
  • billing paused
  • wait / async / support choices

Если у эксперта плохая связь, клиент должен видеть, что услуга защищена и кредиты не списываются молча.

Required

5. Expert unavailable / delayed

  • offline/busy after selection
  • available alternative experts
  • free-minute/bonus offer if used
  • ETA or waiting state
  • send async / choose another expert
  • refund/support hint if fulfillment fails

Нельзя строить запуск на допущении, что эксперт всегда online.

Required

6. Connecting to expert / no-charge progress

  • selected expert
  • connection progress
  • expected wait, e.g. 15-20 seconds if true
  • explicit no-charge during connection
  • keep waiting / choose another expert / support fallback

Запись показывает это как отдельный trust state: клиент не должен думать, что его уже тарифицируют во время подключения.

Required

7. Low credits / no credits

  • low balance warning
  • session stopped state
  • top up and continue
  • end and save transcript
  • auto-refill offer with skip/standard payment

Самый критичный interruption в paid chat.

Required

8. Failed payment / retry another method

  • payment failed message
  • selected package/order summary
  • try another method
  • choose another package
  • safe return to chat or support

Это revenue-critical state: покупатель не должен застрять на продолжении консультации.

Required

9. Failed send/upload

  • message sending/failed/retry
  • attachment uploading/failed/retry/remove
  • unsupported file

Достаточно component state set, не отдельные большие страницы.

Required

10. Ended consultation / transcript

  • consultation ended
  • transcript saved
  • question summary and final answer
  • follow-up, rating, save/bookmark

Закрывает платное обещание и создает следующий заказ.

Required

11. Support/refund/report

  • report message/expert
  • request help/refund
  • submitted/in review/resolved

Нужен понятный safety/support path внутри платного чата.

Не просим сейчас

Defer

Messenger polish

Archive/mute/delete, full notification center, full search mechanics, edited/deleted-for-everyone messages, deep saved-message management.

Не блокирует первый paid consultation launch.

Defer

Complex extras

Scheduling flow, advanced moderation copy, complex media gallery, full voice recording lifecycle if voice is not MVP.

Можно вернуться после MVP-flow.

Короткий brief Антону: “Базовый desktop chat уже есть. Не редизайним весь messenger. Нужно дорисовать только paid-service states: offer/payment/credits, session status, question intake, connecting to expert with no-charge progress, expert connection lost/bad connection, expert busy/unavailable fallback with alternative experts, low/no credits top-up, failed payment/retry, failed send/upload, ended transcript, support/refund/report.”
